1. Жизненный цикл объекта

5 этапов:

1. Выделение памяти
2. Инициализация (вызов конструктора)
3. Использование
4. Уничтожение (вызов деструктора)
5. Освобождение памяти


2. Конструктор

Что это: специальный метод, который вызывается при создании объекта

Особенности:

1. Имя совпадает с именем класса
2. Нет возвращаемого типа (даже void)
3. Вызывается автоматически
4. Нужен для инициализации полей

Виды конструкторов:

1. По умолчанию (без параметров)
2. Параметризированный (с параметрами)
3. Можно перегружать (несколько конструкторов в одном классе)


3. Список инициализации

Синтаксис:
Класс::Класс(параметры) : поле1(значение), поле2(значение) { }

Преимущества:

1. Более эффективная инициализация
2. Обязателен для:
  1. const полей
  2. Ссылок
  3. Объектов без конструктора по умолчанию


4. Деструктор

Что это: специальный метод, вызываемый при уничтожении объекта

Особенности:

1. Имя: ~ + имя класса
2. Нет параметров
3. Нет возвращаемого типа
4. Может быть только один (не перегружается)
5. Вызывается автоматически

Когда вызывается:

1. При выходе объекта из области видимости
2. При удалении через delete
3. При завершении программы (для глобальных объектов)

Важно: Если класс использует динамическую память, деструктор обязан её освобождать!


5. Динамическая память в классах

Локальные объекты:

1. Создаются: Book b;
2. Уничтожаются автоматически при выходе из области видимости

Динамические объекты:

1. Создаются: Book× b = new Book();
2. Удалять вручную: delete b; (вызовет деструктор)


6. Порядок вызова

При создании:

1. Выделяется память
2. Вызывается конструктор

При уничтожении:

1. Вызывается деструктор
2. Освобождается память

Для полей-объектов:

1. Конструкторы: в порядке объявления полей
2. Деструкторы: в обратном порядке


7. Главные выводы

1. Конструктор = настройка объекта при рождении
2. Деструктор = уборка за объектом перед смертью
3. Если класс использует new - нужен delete в деструкторе
4. Список инициализации делает код эффективнее
