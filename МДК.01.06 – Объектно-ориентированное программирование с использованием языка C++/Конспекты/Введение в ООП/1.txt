 1. Основные парадигмы программирования
1. Процедурный подход: данные и функции существуют отдельно, функции работают с данными
2. Объектно-ориентированный подход: данные и функции объединяются в объекты

 2. Процедурный подход
1. Данные и функции разделены
2. Часто используются глобальные переменные
3. Недостатки:
 1. Глобальное состояние
 2. Отсутствие контроля доступа к данным
 3. Сложность масштабирования
 4. Трудности поддержки крупных систем

 3. Объектно-ориентированный подход (ООП)
1. Данные и функции объединяются в единую сущность
2. Программа строится вокруг объектов
3. Моделирует реальные объекты предметной области
4. Основные принципы ООП:
 1. Инкапсуляция
 2. Абстракция
 3. Наследование
 4. Полиморфизм

 4. Класс и объект
1. Класс - пользовательский тип данных, объединяющий:
 1. Поля (данные)
 2. Методы (функции для работы с данными)
2. Объект - экземпляр класса
 1. Имеет собственное состояние
 2. Использует общий набор методов класса

 5. Объявление класса в C++

class ClassName {
  // поля и методы
};

 Пример создания объекта:
ClassName objectName;


6. Инкапсуляция

 Механизм сокрытия внутреннего состояния объекта
 Предоставление контролируемого доступа через методы
 Модификаторы доступа в C++:
  public - доступен всем
  private - доступен только внутри класса
  protected - доступен внутри класса и наследникам

7. Пример инкапсуляции

 Прямой доступ к полю (например, balance) запрещён
 Изменение состояния только через методы
 Контроль корректности операций

8. Несколько объектов одного класса

 Каждый объект имеет своё состояние
 Методы класса общие, но работают с конкретным объектом
