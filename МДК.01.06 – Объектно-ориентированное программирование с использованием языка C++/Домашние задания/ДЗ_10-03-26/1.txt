1. Копирование объектов

В C++ объекты копируются при передаче по значению, возврате из функций и инициализации одного объекта другим.

2. Поведение компилятора по умолчанию

Если класс содержит только простые типы, автоматическое копирование работает корректно. Если класс управляет динамической памятью, возникает проблема поверхностного копирования.

3. Описание проблемы

Компилятор копирует значения полей, включая указатели. В результате два объекта указывают на одну область памяти, и при уничтожении деструктор вызывается дважды, что приводит к двойному освобождению памяти и неопределённому поведению.

4. Поверхностное и глубокое копирование

При поверхностном копировании копируются только значения полей, включая адреса указателей. При глубоком копировании создаётся новая область памяти, и данные копируются в неё. Для корректной работы класса с ресурсами необходимо реализовать глубокое копирование.

5. Копирующий конструктор

Это специальный конструктор, который принимает ссылку на объект того же типа. Он используется для создания нового объекта как копии существующего.

6. Оператор копирующего присваивания

Копирование может происходить не только при инициализации, но и при присваивании. В этом случае вызывается оператор присваивания. Если его не определить, компилятор создаст поверхностную версию.

При реализации оператора присваивания необходимы: проверка самоприсваивания, освобождение старого ресурса, копирование данных и возврат ссылки на текущий объект.

7. Правило трёх

Если класс содержит деструктор и управляет ресурсом, то необходимо реализовать деструктор, копирующий конструктор и оператор копирующего присваивания. Игнорирование этого правила приводит к ошибкам управления памятью.

8. Когда вызывается копирующий конструктор

Он вызывается при инициализации одного объекта другим, при передаче по значению и при возврате объекта из функции.

9. Запрет копирования

В некоторых случаях копирование объекта может быть нежелательным, и его можно запретить.
