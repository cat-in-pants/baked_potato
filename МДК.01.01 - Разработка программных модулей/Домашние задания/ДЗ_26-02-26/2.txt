1. Существующие слои проекта
В учебном проекте можно выделить:
Слой представления (UI) - ввод/вывод данных
Слой бизнес-логики - обработка правил и вычисления
Слой данных - хранение и получение информации
Точка входа (main) - запуск приложения
2. Зоны ответственности
main - запуск программы
UI - взаимодействие с пользователем
Логика - обработка данных
Модели - структура данных
Хранилище - доступ к данным
3. Зависимости
main зависит от всех модулей
UI напрямую вызывает бизнес-логику
Логика может напрямую работать с данными
Отсутствуют интерфейсы и абстракции
4. Архитектурные проблемы
смешение UI и бизнес-логики
жёсткие зависимости между модулями
один класс выполняет несколько задач
сложность тестирования
5. Предложения по улучшению
разделить проект на папки: presentation, service, repository, domain
изолировать бизнес-логику от ввода/вывода
внедрить зависимости через конструктор
убрать прямой доступ к данным из UI
повысить связность и снизить зацепление
