1. Назначение библиотеки <chrono>

Замеры времени для сравнения алгоритмов и оптимизации кода
Объективная оценка производительности вместо субъективных ощущений
Замена устаревшей функции clock(), которая имеет проблемы с точностью и кроссплатформенностью

2. Ключевые понятия библиотеки

2.1. Часы (Clocks) - источники времени

system_clock - системное время (может изменяться пользователем)
steady_clock - монотонные часы (идеальны для замеров производительности)
high_resolution_clock - часы с максимальным разрешением

2.2. Временная точка (Time Point)

Конкретный момент времени, полученный от часов

2.3. Длительность (Duration)

Интервал между двумя временными точками
Имеет единицы измерения (наносекунды, микросекунды, миллисекунды, секунды)

3. Преимущества <chrono>

Типобезопасность - компилятор проверяет корректность операций с временем
Кроссплатформенность - единый API для разных операционных систем
Высокая точность - использование аппаратных таймеров системы

4. Основные операции

1. Получение текущего времени - clock::now()
2. Вычисление длительности - разность двух временных точек
3. Приведение к нужным единицам - duration_cast<unit>()
4. Получение численного значения - duration.count()
