1. Цель применения стиля кода

 Для компьютера стиль кода не имеет значения
 Для разработчиков хороший стиль:
  Ускоряет понимание кода
  Снижает количество ошибок
  Упрощает поддержку и доработку
  Сохраняет нервы команды


2. Ключевой принцип написания кода

Код нужно писать так, будто следующий разработчик - это ты через полгода


3. Именование

Основные правила:

 Имя должно объяснять смысл, а не экономить символы
 Нужно использовать единый стиль именования в проекте

Стили именования:

 camelCase - для функций и переменных
 PascalCase - для типов данных, классов
 snake_case - иногда применяется в проектах, особенно для констант


4. Использование констант

 Ключевое слово const применяется везде, где значение не должно меняться
 Это повышает читаемость и предотвращает случайные изменения


5. Области видимости

 Следует минимизировать области видимости переменных
 Глобальные переменные нужно избегать, так как они:
  Усложняют отладку
  Затрудняют тестирование
  Ухудшают масштабируемость кода

Альтернативы глобальным переменным:

 Передача параметров в функции
 Использование возвращаемых значений
 Применение классов и объектов


6. Структура функций

 Функция должна выполнять одну конкретную задачу
 Рекомендуемый размер функции - не более 30–40 строк
 Если функцию сложно объяснить словами, её следует упростить или разделить


7. Использование STL (Standard Template Library)

 Следует использовать стандартные контейнеры и алгоритмы
 Это уменьшает количество ручного кода и повышает надежность


8. Минимизация зависимостей

 Нужно подключать только необходимые заголовочные файлы (#include)
 Избыточные зависимости замедляют компиляцию и увеличивают связность кода


9. Обработка ошибок и предупреждений

 Все возможные ошибки нужно обрабатывать явно
 Предупреждения компилятора не следует игнорировать - они часто указывают на потенциальные проблемы


10. Комментарии

Комментарии нужны:

 Для объяснения сложной логики
 Для описания неочевидных решений
 В заголовках функций и классов

Комментарии не нужны:

 Для очевидного кода
 Для повторения того, что ясно из имен переменных


11. Форматирование кода

Единые правила форматирования:

 Отступы - используются последовательно (пробелы или табуляция)
 Переносы строк - длинные строки разбиваются
 Пробелы - расставляются вокруг операторов, после запятых

Хорошее форматирование улучшает читаемость и уменьшает визуальный шум
